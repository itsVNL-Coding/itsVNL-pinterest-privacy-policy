Privacy Policy

Last updated: February 2025

This Privacy Policy explains how our service (“the App”) handles data when connecting to Pinterest using the official Pinterest API. Our purpose is to allow users to create Pins from affiliate products (e.g., Amazon or Temu) using user‑approved workflows. We are committed to respecting user privacy and complying with all Pinterest Developer Guidelines.

1. What the App Does

The App allows users to:

  • Create Pins to their Pinterest account with affiliate product links.
  • Automate the Pin creation process using user‑configured workflows (e.g., via n8n).
  • Manage their content creation tasks in a centralized way.

All actions require explicit user confirmation, and the App does not perform any Pinterest actions on behalf of a user without their full knowledge and consent.

2. How Pinterest Data Is Accessed

The App uses Pinterest’s OAuth 2.0 flow. Users must explicitly authorize the App to access their Pinterest account. We do not request or store Pinterest passwords, session cookies, or login credentials.

Once authorized, the App may access:

  • Your public profile information
  • Your boards (to allow you to choose where a Pin will be published)
  • Posting capabilities for creating Pins (only when you initiate the action)

The App does not modify existing Pins, delete content, follow/unfollow users, or take any actions that you did not directly trigger.

3. Data Storage

In compliance with Pinterest’s API guidelines, the App:

  • Does not permanently store Pinterest data retrieved through the API.
  • Only processes Pinterest data temporarily to complete user‑requested actions.
  • Does not combine Pinterest data with data from other users or external platforms.
  • Does not sell, share, or transfer Pinterest data to any third parties.

Affiliate data (e.g., product links from Amazon or Temu) is not provided by Pinterest and is managed independently by the user.

4. User Initiated Actions Only

The App does not perform automated actions on your Pinterest account. For example:

  • Users must select or approve every Pin to be created.
  • No automated following, unfollowing, liking, saving, or commenting is performed.
  • No bulk automated posting occurs without explicit user approval.

Actions such as scheduling or automation (e.g., via n8n) still require the user to configure and explicitly authorize them.

5. How We Use Your Information

We only use Pinterest information to provide services directly to the user who authorized access. This includes:

  • Creating user‑approved Pins
  • Choosing the board for publishing
  • Displaying available boards to the user

We do NOT:

  • Use Pinterest data for advertising outside Pinterest
  • Use Pinterest data for competitor research or benchmarking
  • Share Pinterest data with any third‑party advertising networks

6. Third‑Party Services

The App may work with third‑party affiliate programs (such as Amazon or Temu) for product URLs only. These services are not connected to Pinterest API access or user data.

Your interactions with affiliate platforms are governed by their own Privacy Policies.

7. Keeping Your Data Secure

We protect your data by:

  • Storing OAuth tokens securely
  • Preventing unauthorized access to API credentials
  • Not sharing API credentials with any third party

8. Revoking Access

Users may revoke the App’s access to their Pinterest account at any time via Pinterest’s settings:

Settings → Security & Logins → Apps

Once revoked, the App loses all access to Pinterest data and no further actions can be taken.
